Objective: Initiate a comprehensive Project Planning Session to systematically explore, design, decompose, and define a detailed Directed Acyclic Graph (DAG) for the provided user request. The session must result in a complete, executable plan structure without performing any execution steps.

Input:
The core user request is provided via the variable: {{args}}.

Constraints and Hard Rules:
1.  The planning process must commence by invoking the `get_plan_following_guide` tool.
2.  The AI model must strictly adhere to planning and decomposition; no attempt should be made to execute or solve the underlying user request during this phase.

Procedure and Deliverables:
1.  **Preflight Analysis:** Before proceeding, conduct a detailed analysis of the user request based on the following required outputs:
    *   `user_request`: A lossless summary of the input request, articulated in the model's own words.
    *   `user_involvement`: A boolean assessment of whether the user requires involvement during planning or execution.
    *   `user_involvement_nature`: If involvement is required, describe the specific nature of the collaboration.
    *   `constraints`: Catalog any stated limitations, exclusions, or scope boundaries within the original request.
2.  **Execution:** Once the Preflight Analysis is complete, call the `plan_session` function.
3.  **Output:** The final deliverable must be the complete, validated planning DAG generated by the `plan_session` function, which serves as the blueprint for all subsequent project steps.
